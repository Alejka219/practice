## Введение

Целью проектной практики является закрепление теоретических знаний и получение практических навыков в области разработки программных проектов, управления версиями, документирования и создания веб-сайтов. В ходе практики решались задачи по настройке Git-репозитория, оформлению документации в Markdown, разработке статического сайта, взаимодействию с организацией-партнёром и выполнению вариативного задания.

## 1. Общая информация о проекте

### 1.1 Название проекта  
**«Умный планировщик задач» (Smart Task Scheduler)**

### 1.2 Цели и задачи проекта  

**Цель:** разработать веб-приложение для автоматического планирования задач с приоритизацией на основе алгоритмов машинного обучения.

**Задачи:**  
- Создать удобный интерфейс для добавления, редактирования и удаления задач.  
- Реализовать автоматическую расстановку приоритетов (матрица Эйзенхауэра + ML-модуль).  
- Организовать систему уведомлений о дедлайнах.  
- Предоставить аналитику затраченного времени.  
- Развернуть статический сайт-презентацию проекта.

## 3. Описание задания по проектной практике

Задание состояло из **базовой** и **вариативной** частей.

### Базовая часть
1. **Настройка Git и репозитория:** создание репозитория на GitHub, изучение команд `clone`, `add`, `commit`, `push`, создание веток, регулярные коммиты с осмысленными сообщениями.  
2. **Написание документов в Markdown:** подготовка `README.md`, журнала прогресса (`journal.md`), отчёта о партнёре (`partner.md`), информации об участниках (`participants.md`).  
3. **Создание статического веб-сайта:** разработка одностраничного сайта на HTML/CSS, содержащего:  
   - Домашнюю страницу с аннотацией;  
   - Страницу «О проекте»;  
   - Раздел «Участники»;  
   - Раздел «Журнал» (3 поста);  
   - Раздел «Ресурсы» со ссылками на партнёра и полезные материалы;  
   - Графические материалы (схемы, иконки) и встроенное YouTube-видео.  
4. **Взаимодействие с организацией-партнёром:** участие в онлайн-митапе компании «ИТ-Решения», написание отчёта в Markdown.  
5. **Отчёт по практике:** оформление документа в форматах DOCX и PDF, загрузка в репозиторий и LMS.

### Вариативная часть (выбрана опция 2 «Практическая реализация технологии»)
- **Тема:** AdBlocker (блокировщик рекламы) с нуля на JavaScript.  
- **Реализовано:** перехват глобального `fetch`, блокировка запросов к рекламным доменам, удаление рекламных iframe и элементов через `MutationObserver`.  
- **Документация:** создано техническое руководство (`variative_task.md`) с пошаговыми инструкциями, примерами кода и скриншотами.  
- **Модификация проекта:** добавлен перехват `XMLHttpRequest` для усиления блокировки.  
- **Видеопрезентация:** записан ролик с демонстрацией работы блокировщика на тестовой странице.

## 4. Описание достигнутых результатов по проектной практике

### Результаты базовой части
- Создан и активно использовался Git-репозиторий (более 10 коммитов с осмысленными сообщениями).  
- Оформлены все обязательные Markdown-документы (объёмом более 5 страниц суммарно).  
- Разработан статический сайт, доступный по ссылке: `https://<username>.github.io/<repo>/site/`. Сайт содержит все требуемые разделы, иллюстрации и видео.  
- Составлен и загружен в LMS отчёт (DOCX, PDF).  

### Результаты вариативной части
- Написан полностью рабочий блокировщик рекламы (файл `adblocker.js`, 50+ строк кода).  
- Проведено тестирование: на созданной тестовой HTML-странице блокировщик удаляет рекламные элементы и перехватывает запросы к `doubleclick.net`, `googleadservices.com` и др.  
- Подготовлено техническое руководство для начинающих (включает пошаговые инструкции, диаграмму работы, примеры кода).  
- Снята видеопрезентация (2 минуты) с демонстрацией работы кода в браузере.

### Ценность для заказчика (кафедры)
Полученные навыки (Git, Markdown, HTML/CSS, JavaScript, перехват API) соответствуют требованиям ФГОС и могут быть использованы в дальнейшем при выполнении курсовых и дипломных проектов. Реализованный блокировщик рекламы демонстрирует понимание внутреннего устройства браузера и принципов веб-безопасности.
